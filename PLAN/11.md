# PLAN 11：下一阶段架构升级执行方案

## 结论

当前状态：

- Phase 1 Hybrid Memory：已完成
- Phase 2 LangGraph 图化：已完成第一版
- Phase 3 MCP：暂缓

这三项能力值得做，但不应同时启动。

当前最优顺序：

1. 先做 Memory Layer：把现有 SQLite 记忆 + 知识三元组升级为 Hybrid Memory。
2. 再做 LangGraph：把现有手写自主循环逐步改造成显式状态图。
3. 最后评估 MCP：只有在需要统一接入多外部系统时再启动。

原因不是“概念先进”，而是“和当前代码的衔接成本最低、收益最大”。

## 当前状态判定

### 1. Memory Layer：部分已有，但不是 GraphRAG

当前仓库已经具备图谱记忆雏形：

- openclaw_continuity.py 中已有 knowledge_triples 表。
- 已支持从学习结果中提取 triples 并写入持久层。
- recall_my_memories 已经把知识三元组和普通记忆碎片一起返回。
- 还存在一个可选 Neo4j 桥接，用于历史快照增强。

但当前缺失真正的向量层：

- 没有 embedding 生成链路。
- 没有向量索引或向量数据库。
- 没有向量召回 + 图谱召回的混合排序。
- 没有基于实体关系扩展上下文的 GraphRAG 检索流程。

结论：当前是“图谱化记忆雏形”，不是 GraphRAG。

### 2. MCP：当前未接入

当前工具系统是：

- 内置工具 + workspace/tools 目录下的动态工具。
- 通过 @tool 和本地扫描加载。
- 由 create_react_agent 使用。

当前并不存在：

- MCP client
- MCP server 注册/发现
- 工具协议协商
- 基于 MCP 的统一外部连接层

结论：当前没有正式 MCP 集成。

### 3. LangGraph：已基础接入，但尚未图化主循环

当前仓库已经使用 create_react_agent，说明 LangGraph 已经进入执行链路。

但当前 autonomy_loop 仍然是手写 Python 心跳循环，反思、调度、门控逻辑并没有被建模成显式图节点与边。

结论：当前是“LangGraph agent 已接入”，不是“LangGraph-native 自主循环系统”。

## 决策

本阶段正式启动 Memory Layer 升级，不同时推进 MCP，也不立刻重写 autonomy_loop 为完整 StateGraph。

换句话说：

- 现在要做：Hybrid Memory
- 下一阶段做：LangGraph 图化
- 暂缓：MCP

## Phase 1：Hybrid Memory

### 目标

把当前记忆系统从：

- 文本记忆
- 知识三元组
- 可选 Neo4j 快照

升级为：

- Episodic Memory：原始事件和摘要
- Graph Memory：实体关系和三元组
- Vector Memory：语义相似召回
- Hybrid Recall：关键词、向量、图谱联合召回

### 这一步为什么先做

因为它直接增强：

- 长期记忆命中率
- 模糊问题下的召回能力
- 反思时的上下文质量
- 后续 LangGraph 节点中的检索质量

而且这一步能最大复用现有 MemoryCore、recall_my_memories、Neo4jBridge 和学习蒸馏链路。

### 本阶段最小落地范围

第一阶段不追求“最豪华”的 GraphRAG，只追求一条稳定可验证的最小闭环：

1. 为记忆引入向量字段或独立向量表。
2. 新增 embedding 生成函数，先只覆盖 learning / reflection / conversation summary 三类高价值记忆。
3. 新增向量相似召回函数。
4. 将 recall_my_memories 升级为三路召回：
   - 关键词召回
   - 向量召回
   - 三元组召回
5. 对三路结果做简单融合排序，先不用复杂 reranker。

### 实施步骤

#### Step 1. 设计向量存储层

建议先使用 SQLite 内嵌方案，不马上引入重型外部向量数据库。

新增一张表，例如：

- memory_embeddings
  - memory_id
  - source_type
  - content_text
  - embedding_json
  - timestamp

说明：

- 先以 JSON 数组存 embedding，优先验证能力而不是追求极致性能。
- 等召回路径稳定后，再决定是否迁移到 FAISS / Qdrant / pgvector。

#### Step 2. 建立 embedding 生成入口

新增统一 helper，例如：

- _embed_text(text: str) -> List[float]

要求：

- 有明确的空文本保护。
- 有失败降级路径，不能让记忆写入因 embedding 失败而中断。
- 初期只处理有限长度文本，避免极长上下文直接进 embedding。

#### Step 3. 记忆写入时同步生成向量

优先接入以下路径：

- search_and_learn 的 distilled summary
- 自主反思结果
- 对话摘要或用户任务摘要

原则：

- 原始记忆写入始终优先成功。
- embedding 写入失败只能降级，不得阻塞主流程。

#### Step 4. 新增向量召回

新增接口，例如：

- recall_by_vector(query: str, limit: int = 8)

实现要求：

- 将 query 先 embedding。
- 计算与 memory_embeddings 中候选项的相似度。
- 初版允许用余弦相似度 + Python 线性扫描。
- 结果中返回 score，便于后续混合排序。

#### Step 5. 升级 recall_my_memories

将现有 recall_my_memories 升级成统一的 Hybrid Recall 出口。

召回顺序建议：

1. triples：给出明确关系
2. vector hits：给出语义相关内容
3. keyword hits：给出精确命中内容

输出要求：

- 区分不同来源
- 避免重复片段
- 对同一 memory_id 做合并

#### Step 6. 为后续 Neo4j 对接预留边界

本阶段不要重写 Neo4jBridge。

只做一件事：把图谱写入和召回 helper 抽象清楚，让未来可以把 SQLite triples 和 Neo4j graph 做成同一层接口。

## Phase 2：LangGraph 图化

这一阶段先不启动实现，但提前冻结目标，避免第一阶段写偏。

目标不是替换 create_react_agent，而是把 autonomy_loop 的控制流图化。

预期节点：

1. Observe：观察用户状态、事件队列和当前任务
2. Retrieve：调用 Hybrid Memory
3. Decide：判断当前是 THINK 还是 ACTION
4. Execute：执行工具或产出回复
5. Reflect：写入新记忆并评估下一步
6. Retry / Exit：根据结果回退或结束

当前不做，是因为没有高质量检索层时，图化后的收益会被记忆质量拖住。

## Phase 3：MCP

这一阶段只在以下条件满足时再开工：

1. 本地工具数量继续增长，维护成本明显升高。
2. 需要统一接入多个外部系统。
3. 希望把“本地 Python 工具”与“外部系统工具”抽成统一协议边界。

在那之前，MCP 不是当前瓶颈。

## 本轮实际工作范围

本轮开始时，只做以下内容：

1. 设计并接入最小向量存储层。
2. 给高价值记忆写入 embedding。
3. 增加向量召回函数。
4. 把 recall_my_memories 升级为 Hybrid Recall。
5. 保持 openclaw_continuity.py 继续可编译、可启动、可降级运行。

本轮不做：

1. 不引入 MCP。
2. 不全面重写 autonomy_loop。
3. 不引入重型外部向量数据库，除非 SQLite 方案证明明显不足。
4. 不做与本轮目标无关的大规模拆文件。

## 目标文件

- openclaw_continuity.py
- 如有必要，可新增一个轻量级记忆 helper 文件，但优先先在现有主文件内闭环验证。

## 验收标准

### 第一阶段完成标准

1. 可以把高价值记忆写入向量层。
2. recall_my_memories 在输入模糊描述时，能返回语义相关记忆，而不只依赖关键词。
3. recall_my_memories 同时保留 triples 关系输出。
4. embedding 失败不会导致主流程中断。
5. openclaw_continuity.py 继续通过编译检查。
6. 脚本启动后，记忆检索路径不引入新的启动阻塞。

## 开工顺序

从现在开始，第一刀不是 MCP，也不是 LangGraph 重写。

第一刀是：

- 给 MemoryCore 增加向量存储和向量召回接口。

完成这个闭环后，再继续把 recall_my_memories 升级成真正的 Hybrid Recall。
