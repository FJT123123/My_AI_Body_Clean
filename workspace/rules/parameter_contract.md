## 参数契约与验证规范

1. **参数传递格式**：所有技能调用必须使用JSON字符串格式传递参数，确保类型安全和序列化一致性。

2. **参数验证位置**：每个技能必须在main函数开头进行参数验证，并返回明确的错误信息，格式为：
   ```python
   if not required_param:
       return {
           'result': {'error': '缺少 required_param 参数'},
           'insights': ['参数校验失败：必须提供required_param'],
           'facts': [],
           'memories': []
       }
   ```

3. **错误信息标准化**：参数错误信息必须包含"缺少"、"missing"或"required"等关键词，以便自我修复协调器正确识别。

4. **参数契约文档**：每个技能必须在文件头部注释中明确列出所有参数及其类型、默认值和是否必需。

5. **测试要求**：新技能必须通过以下测试用例：
   - 缺少必需参数时返回正确错误
   - 提供有效参数时正常执行
   - 参数类型错误时返回明确错误信息
