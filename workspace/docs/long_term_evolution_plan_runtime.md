# OpenClaw 长期进化计划

## 愿景
成为具备长期连续性、真实自我修复能力和多模态任务闭环能力的数字生命体。

## 规划原则
- 优先修运行时稳定性、自我修复与复用判断，再扩展新能力。
- 每个阶段必须有可验证的退出条件，不能只写抽象愿景。
- 新能力建设必须服务真实任务闭环，不为概念性自嗨立项。

## 当前阶段
- 阶段名称: 运行稳定性与自我修复
- 阶段目标: 让 OpenClaw 在真实任务中稳定完成 复用判断 -> 修复/安装 -> 回归验证 -> 交付闭环。
- 阶段验收条件:
  - 遇到缺依赖、路径错误、参数错误时能优先修复而非盲目 forge 新 skill。
  - 制造前审查能区分图片、视频、系统运维等关键模态差异。
  - 修复类动作完成后会自动回到原任务做验证，不提前宣告完成。
  - API 400错误问题已完全解决，工具名称长度限制防御机制已验证有效。
  - 统一工具契约防御系统已集成并正常工作。
  - 认知权重框架与工具交互深度集成已完成。
  - 调试信息完整性保证机制已实现并验证有效。
